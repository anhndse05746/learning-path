Step 5: Deepen DevOps & Deployment Knowledge

Goal

Own the full lifecycle of a backend app, from development to deployment and monitoring.

Duration

4 weeks (assuming 2-3 hours/day, 5 days/week)

Theory (What to Learn)

  1. CI/CD Pipelines
    • Continuous Integration and Deployment concepts.
    • Tools: GitHub Actions, Jenkins.
  2. Containerization
    • Docker basics: Images, containers, Dockerfile.
    • Orchestration with Docker Compose.
  3. Cloud Deployment
    • AWS/GCP services (e.g., EC2, S3, Lambda).
    • Serverless architecture.
  4. Monitoring & Logging
    • Tools: PM2, New Relic, custom logging.
    • Metrics: CPU, memory, request latency.

Hands-On Practice Plan

Week 1: Containerization Basics

  • Day 1: Theory
    • Read: Docker Get Started
  • Day 2-3: Exercise 1 - Dockerize an App
    • Task: Dockerize a simple Node.js API (e.g., from Step 2). Run it locally.
    • Example Dockerfile: 
      FROM node:16
WORKDIR /app
COPY package*.json ./
RUN npm install
COPY . .
CMD ["node", "index.js"]
  • Day 4-5: Exercise 2 - Docker Compose
    • Task: Use docker-compose.yml to run your API with a MongoDB container.
    • Example: 
      version: '3'
services:
  app:
    build: .
    ports:
      - "3000:3000"
  mongo:
    image: mongo
    ports:
      - "27017:27017"

Week 2: CI/CD Setup

  • Day 1: Theory
    • Read: GitHub Actions Docs
  • Day 2-3: Exercise 1 - Basic Pipeline
    • Task: Set up a GitHub Actions workflow to run tests on push.
    • Example .github/workflows/test.yml: 
      name: Test
on: [push]
jobs:
  test:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v3
      - uses: actions/setup-node@v3
        with:
          node-version: '16'
      - run: npm install
      - run: npm test
  • Day 4-5: Exercise 2 - Deploy Pipeline
    • Task: Extend the pipeline to build and deploy your Docker image to a registry (e.g., Docker Hub).

Week 3: Cloud Deployment

  • Day 1: Theory
    • Read: AWS Getting Started
  • Day 2-3: Exercise 1 - AWS EC2
    • Task: Deploy your Dockerized app to an AWS EC2 instance using SSH and Docker.
  • Day 4-5: Exercise 2 - Serverless
    • Task: Deploy a simple Node.js function (e.g., API endpoint) to AWS Lambda.

Week 4: Monitoring & Optimization

  • Day 1: Theory
    • Read: PM2 Monitoring Docs
  • Day 2-3: Exercise 1 - PM2 Setup
    • Task: Run your app with PM2, enable monitoring, and restart on crash.
    • Example: 
      pm2 start index.js --watch
  • Day 4-5: Exercise 2 - Custom Logging
    • Task: Add Winston or a custom logger to your app. Log requests and errors to a file.

Milestones

  • End of Week 2: Automate testing and deployment with CI/CD.
  • End of Week 4: Deploy and monitor a production-ready app.
