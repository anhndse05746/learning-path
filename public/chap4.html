Step 4: Level Up Testing

Goal

Write comprehensive, reliable tests to ensure production-ready code.

Duration

4 weeks (assuming 2-3 hours/day, 5 days/week)

Theory (What to Learn)

  1. Testing Types
    • Unit tests: Individual functions/modules.
    • Integration tests: Combined components.
    • End-to-end (E2E) tests: Full workflows.
  2. Testing Frameworks
    • Jest for unit/integration testing.
    • Supertest for API testing.
    • Mocking dependencies.
  3. Test-Driven Development (TDD)
    • Write tests first, then code.
    • Red-Green-Refactor cycle.

Hands-On Practice Plan

Week 1: Unit Testing Basics

  • Day 1: Theory
    • Read: Jest Getting Started
  • Day 2-3: Exercise 1 - Simple Unit Tests
    • Task: Write unit tests for a utility module (e.g., math functions) using Jest.
    • Example: 
      // math.js
function add(a, b) { return a + b; }
module.exports = { add };

// math.test.js
const { add } = require('./math');
test('adds 1 + 2 to equal 3', () => {
  expect(add(1, 2)).toBe(3);
});
  • Day 4-5: Exercise 2 - Mocking
    • Task: Test a function that calls an external API by mocking the API response with Jest.

Week 2: Integration Testing

  • Day 1: Theory
    • Read: Supertest Docs
  • Day 2-3: Exercise 1 - API Tests
    • Task: Write integration tests for a Node.js API (e.g., GET /users) using Supertest.
    • Example: 
      const request = require('supertest');
const app = require('./app');
describe('GET /users', () => {
  it('responds with json', async () => {
    const res = await request(app).get('/users');
    expect(res.status).toBe(200);
    expect(res.body).toHaveProperty('users');
  });
});
  • Day 4-5: Exercise 2 - Database Integration
    • Task: Test API endpoints with a real database (e.g., MongoDB). Use setup/teardown to reset data.

Week 3: TDD Practice

  • Day 1: Theory
    • Watch: “TDD Basics” by Kent C. Dodds (YouTube)
  • Day 2-3: Exercise 1 - TDD Feature
    • Task: Use TDD to build a user registration endpoint (write tests first).
  • Day 4-5: Exercise 2 - Refactor
    • Task: Refactor an existing untested module, add tests, and ensure 80% coverage (use jest --coverage).

Week 4: Advanced Testing

  • Day 1: Theory
    • Read: Jest Advanced Features
  • Day 2-3: Exercise 1 - Edge Cases
    • Task: Add tests for edge cases (e.g., invalid input, timeouts) in your API.
  • Day 4-5: Exercise 2 - E2E Testing
    • Task: Write an E2E test for a full workflow (e.g., register → login → get user) using Supertest or a tool like Cypress.

Milestones

  • End of Week 2: Write reliable unit and integration tests for a small app.
  • End of Week 4: Achieve 80% test coverage with TDD and E2E tests.
